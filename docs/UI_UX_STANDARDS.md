# UI/UX Standards

## Accessibility

- Compliance: WCAG 2.1 AA
- Features: Keyboard navigation, screen reader support, high contrast modes
- Testing: Automated and manual audits

## Usability Constraints

- Minimal cognitive load: Progressive disclosure, clear hierarchies
- Error Prevention: Confirmations for actions, undo options
- Performance: <2s load times

## Copy Style Guide

- Tone: Professional, empathetic, non-alarming
- Language: Simple, avoid jargon; reading level 6th grade (Flesch-Kincaid)
- Patient Communications: "Please contact your care team if..." not "You must..."
- Accountability: Always show approver name, timestamp, version

## No Dark Patterns

- Transparent actions; no hidden approvals
- Clear indicators for AI-generated vs human-reviewed content

## Consistency

- Design System: Based on Material Design or similar
- Icons: Standard meanings
- Colors: Red for errors, green for success
